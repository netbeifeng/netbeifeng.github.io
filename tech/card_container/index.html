并不是那个Steam™

最近做Web的作业做的头大。

需求是一个类似于「Bangumi番组计划」的电视剧版本 *（或者说是豆瓣） 名字也是不想吐槽。
Steam™ – Streaming Series Memory

还特地要加一个TradeMark的符号，虽然不太明白这个标志是干嘛的，但感觉瞬间有那味了。

回到任务上来，想要做一个跳脱出传统的布局格式，比如图片轮播或者简单的列表就很千篇一律。


如何保证有图片信息的同时，信息的密度又不至于太低呢。

于是想到了backface-visibility这个优雅的CSS属性。

Backface-Visibility

当我们对某一个组件应用 transform: rotateY(180deg) 的样式时，

我们可以看见这个图片竖直方向上的镜像。

像这样：

  RotateY(180deg)  

我们之所以可以看见 card是因为backface-visibility这个属性的默认值是visible，也就是可见的。

那么，当我们把图片的backface-visibility调整为hidden时，再进行旋转，那么该图片就是不可见的了。

这点可以用来实现我们的“卡片”。
